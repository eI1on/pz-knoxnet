title=[B41] Knox Net
description=[h1]Knox Net[/h1]
description=
description=A server-friendly [b]in-game messenger[/b]-direct chats, group chats, unread tracking, optional on-screen notifications when new messages arrive (delivered through the HUD notification stack in [b]ElyonLib[/b]), and [b]shared map locations[/b] woven into conversations.
description=
description=Open [b]KnoxNet[/b] from the [b]Dock Menu[/b] (supplied by [b]ElyonLib[/b]); the dock badge shows your total unread count. When message alerts are enabled in sandbox, ElyonLib raises lightweight toasts beside the vanilla moodles so you notice traffic even when the chat window is closed.
description=
description=[h1]How it works[/h1]
description=[list]
description=[*]Start a [b]DM[/b] by picking an online player, or typing a username (MP: recipient must exist / have used Knox Net at least once).
description=[*]Create [b]group chats[/b] with drafting, member management, and admin actions (sandbox can restrict who may create groups).
description=[*]Scroll history for each chat, load older pages, and mark threads read from the UI.
description=[*]Use [b]Share Location[/b] to attach your current world position to a message (great for meetups and rescues).
description=[*][b]View on map:[/b] click a history message that has a shared location attachment - Knox Net opens the [b]world map[/b] centred on that pin (toggles map if it was closed).
description=[/list]
description=
description=[h1]Singleplayer & multiplayer[/h1]
description=[list]
description=[*][b]Multiplayer:[/b] the server is authoritative - chat metadata and history persist server-side for co-op and dedicated hosts.
description=[*][b]Singleplayer:[/b] runs locally with the same UI and persistence model.
description=[*][b]Timestamps[/b] are stored as unix time and displayed in each player's local timezone.
description=[*][b]Live notifications[/b] (optional) use ElyonLib [b]HudNotify[/b] toasts when a new message lands while the messenger is closed-toggle cadence in sandbox.
description=[/list]
description=
description=[h1]Group features[/h1]
description=[list]
description=[*]Add / remove members, promote or demote admins, temporary mutes, rename, leave, and disband flows.
description=[*]Owner and admin capabilities respect Knox Net sandbox options where configured.
description=[/list]
description=
description=[h1]Sandbox options[/h1]
description=Configure these on the [b]KnoxNet[/b] sandbox page:
description=[list]
description=[*]Maximum message length
description=[*]History page size
description=[*]Maximum stored messages per chat and retain indefinitely
description=[*]Maximum group members / maximum groups per player
description=[*]Who can create groups (everyone, admins only, or disabled)
description=[*]Show live message notifications
description=[*]Message rate limit (seconds between sends; 0 disables)
description=[/list]
description=
description=[h1]Requirements[/h1]
description=[list]
description=[*][b]ElyonLib[/b] - load it [b]before[/b] Knox Net. It registers the Dock Menu icon with unread badges, renders the chat UI using the shared Elyon theme, routes MP messaging, and powers optional HUD notifications via HudNotify.
description=[/list]
description=
description=[h1]Frequently asked questions[/h1]
description=
description=[h2]Does it replace the vanilla chat?[/h2]
description=No - Knox Net is a separate messenger for DMs and groups with its own persistent history.
description=
description=[h2]Will it work on an existing save?[/h2]
description=Yes; enable whenever you want - no dedicated new-save requirement.
description=
description=[h2]Why can't I DM someone on MP?[/h2]
description=The server resolves recipients by roster (they need to have connected once so Knox Net knows the username exists).
description=
